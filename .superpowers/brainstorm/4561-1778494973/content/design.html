Phase 1 MVP 架构方案

小程序直连方案 · 已确认决策

架构

三层：微信小程序（UI + 探测）→ 云端中台（AI + 用户数据）→ 可选 Probe Agent（Phase 2）

目录结构

netprowl/
├── src/
│   ├── pages/
│   │   ├── discovery/      # 发现页（拓扑图 + 扫描）
│   │   ├── history/        # 历史页
│   │   └── chat/           # AI 问诊页
│   ├── components/         # 自定义组件
│   ├── services/           # 探测服务层
│   ├── utils/              # 工具函数
│   └── app.ts
├── package.json
└── project.config.json

核心模块

  • Discovery：扫描入口 + mDNS + UDP SSDP + TCP 探测 + 拓扑图 Canvas
  • History：扫描记录列表 + 对比视图
  • Chat：AI 问诊（DeepSeek function calling）
  • Services：扫描引擎（mDNS/UDP/TCP）+ 存储服务 + 网络感知

数据流

用户点击扫描 → 读取本机 IP + 推断子网 → mDNS 发现 → SSDP 广播 → TCP 端口探测 → 合并结果 → Canvas 渲染拓扑 → 存入 Storage

扫描引擎策略

  • mDNS 优先（Android 正常，iOS 7.0.18+ 走 TCP 兜底）
  • 白名单端口：80, 443, 8080, 8443, 554, 5000, 9000, 49152
  • 并发控制：≤20 socket/轮次，间隔 50ms
  • 超时：2000ms/端口

错误处理

  • iOS mDNS 禁用 → 自动切换 TCP 探测
  • 网络权限拒绝 → 引导开启
  • Storage 满 → 清理最旧记录
  • 扫描超时 → 显示部分结果 + 重试按钮
